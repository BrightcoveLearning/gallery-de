Hinzufügen von Pigeonhole-Fragen und Antworten zu einem Live-Event-Portal-Erlebnis

{{ page.description }}

Pigeonhole kann verwendet werden, um auf einfache Weise ereignisbasierte Interaktivität (Live-Fragen und -Antworten, Umfragen, Umfragen usw.) zu einem Live-Event-Portal-Erlebnis hinzuzufügen. In der Regel wird eine neue Schublade für das Ereignis erstellt, und anschließend werden Sitzungen für die Live-Ereignisse erstellt. Sobald die Sitzung erstellt wurde, wird der Pigeonhole-Einbettungscode in die Live Event-Vorlage eingefügt. Der Einbettungscode bewirkt, dass der Pigeonhole Q & A-Pod neben dem Live-Event-Player angezeigt wird.

Erstellen eines Pigeonhole-Ereignisses und einer Sitzung

Sie sollten ein neues Pigeonhole für Ihr Live-Event erstellen und dann Sitzungen für jedes Video hinzufügen, für das Fragen und Antworten aktiviert sind.

  1. Melden Sie sich bei Ihrem Pigeonhole-Konto an.
  2. Klicken + Pigeonhole hinzufügen und klicken Sie auf das zu verwendende Konto.
  3. Geben Sie ein Veranstaltungsname und Passcode.
  4. Eingeben Start und Ende Termine für die Veranstaltung. Diese Daten steuern, wie lange das Pigeonhole aktiv sein wird.
  5. Wählen Sie a Zeitzone und Benutzerdefiniertes Branding Option (falls zutreffend).
  6. Klicken Fortsetzen.
  7. Nachdem das Ereignis erstellt wurde, wird die Agenda Element wird in der linken Navigation ausgewählt. Klicken Sitzung hinzufügen > Erstelle neu.
  8. Wählen Sie a Sitzungstyp von Fragen und Antworten.
  9. Geben Sie ein Sitzungsname.
  10. Eingeben Start und Ende Datum und Uhrzeit.
  11. Konfigurieren Sie die Q & A-Einstellungen wie angemessen.
  12. Klicken Sie auf Fragen und Antworten hinzufügen. Bestätigen Sie, dass die Sitzung erstellt wurde.
  13. Klicken Mehr Einstellungen in der linken Navigation.
  14. Drücke den Integrationen Registerkarte oben auf der Seite.
  15. Scrollen Sie nach unten zu Andere Sektion.
  16. Lokalisieren Brightcove Gallery Embed und klicken Sie auf Integrieren.
  17. (Optional) Wenn Ihre Galerieerfahrung mit SSO gesichert wird, wählen Sie Ermöglichen Sie die automatische Anmeldung von Teilnehmern mithilfe von SAML-Attributen. Dadurch werden der Name und die E-Mail-Adresse des Benutzers abgerufen und automatisch bei Pigeonhole Live angemeldet, wenn das Pigeonhole Live-Widget geladen wird. Wenn dies nicht aktiviert ist, werden Benutzer in Pigeonhole als anonym angezeigt. Klicken speichern.
  18. Wählen Sie im Dialogfeld "Brightcove Gallery Embed" Ihre Q & A-Sitzung aus der Dropdown-Liste aus.
  19. Klicken Holen Sie sich Einbettungscode.
  20. Klicken Sie auf den Einbettungscode und kopieren Sie ihn in die Zwischenablage. Dies ist der Code, der in die Galerie eingebettet wird.
  21. Klicken Schließen.

Hinzufügen einer Schublade zu einer Erfahrung

Der Einbettungscode für die Schublade muss zu einer Erfahrung hinzugefügt werden, damit eine Schublade angezeigt wird. Führen Sie die folgenden Schritte aus, um den Einbettungscode zu einer Erfahrung hinzuzufügen.

  1. Öffnen Sie das Galerie-Modul.
  2. Klicken Sie zum Bearbeiten auf Live Event Experience.
  3. Klick auf das Twitter-Einstellungen Verknüpfung.
  4. In dem Twitter-Feed Abschnitt, fügen Sie den Pigeonhole-Einbettungscode ein.
  5. (Optional) Fügen Sie den Einbettungscode in das ein Wohnen und Nach dem Event Seiten auch.
  6. Speichern und veröffentlichen Sie die Änderungen.

Übergabe von SSO-Werten an Gallery

Wenn Ihre Galerieerfahrung mit SSO gesichert wird und Sie die ausgewählt haben Ermöglichen Sie die automatische Anmeldung von Teilnehmern mithilfe von SAML-Attributen Option Beim Erstellen Ihrer Sitzung müssen einige Dinge getan werden, um sicherzustellen, dass die SSO-Attribute für Pigeonhole verfügbar sind. Wenn die SSO-Informationen nicht verfügbar sind, werden Benutzer in Pigeonhole als anonym angezeigt.

  1. Öffnen Sie das Galerie-Modul.
  2. Klicken die Einstellungen in der Kopfzeile.
  3. Klicken Sie in der linken Navigation auf Zugriffskontrollprofile.
  4. Klicken Sie auf das Bearbeitungssymbol für das Profil, das zum Sichern der Erfahrung verwendet wird.
  5. Stellen Sie sicher, dass die Einstellung Alle Attribute anzeigen aktiviert ist.

Sie müssen auch sicherstellen, dass die richtigen SSO-Attribute zugeordnet sind, damit Pigeonhole auf sie zugreifen kann. Pigeonhole sucht nach folgenden Werten:

  • emailaddress
  • name

Das folgende Beispiel zeigt, wie diese Attribute auf der Okta SSOAdmin-Seite verfügbar gemacht werden.

Wenn alles richtig konfiguriert ist, werden die Benutzerinformationen in Pigeonhole angezeigt, wenn sich der Betrachter bei der Erfahrung anmeldet.
