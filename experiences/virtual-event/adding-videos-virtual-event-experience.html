Hinzufügen von Videos zu einem virtuellen Event-Erlebnis

{{ page.description }}

Mit dem Galerie-Sammlungseditor können Sie die Videos auswählen, die in einem virtuellen Ereignis angezeigt werden sollen. Wie andere Erfahrungen vom Typ Portal sind Videos in Sammlungen organisiert.

Klicken Sie auf, um Videos zu einer virtuellen Ereigniserfahrung hinzuzufügen VIDEOS in der linken Navigation.

Organisation der Videosammlung

Die Reihenfolge der Sammlungen steuert, wie sie auf der Startseite des virtuellen Ereignisses angezeigt werden.

Hinweis: Informationen zum Erstellen von Sammlungen und zum Hinzufügen von Videos zu einer Sammlung finden Sie unter Hinzufügen von Videos zu einem Portal Experience.

Die Vorlage für virtuelle Ereignisse enthält vier Hauptbereiche, in denen Videos angezeigt werden, wie auf der Beispielsite angegeben:

  1. Ausgewählte Sammlung - Wird normalerweise für die Live- oder Live-Funktion verwendet simulativer Inhalt das wird während der gesamten Veranstaltung angezeigt. Dies sollte idealerweise basierend auf der Go-Live-Zeit von frühestens bis spätestens bestellt werden, um die beste Erfahrung zu erzielen.
  2. Primärsammlung - Kann alles enthalten, wird jedoch normalerweise verwendet, um bestimmte VoD-Ausbrüche oder Inhalte zu kennzeichnen, auf die Sie zusätzliche Aufmerksamkeit lenken möchten
  3. Sekundärsammlung - Kann alles enthalten, wird jedoch normalerweise verwendet, um bestimmte VoD-Ausbrüche oder Inhalte zu kennzeichnen, auf die Sie zusätzliche Aufmerksamkeit lenken möchten
  4. Spuren - Wird normalerweise für VoD-Inhalte verwendet
Hinweis: Die Sammlungen Featured, Primär und Sekundär unterstützen maximal 24 Videos.
Hinweis: Fügen Sie der Konfigurationsdatei ein statisches Inhaltsobjekt hinzu, um einen Titel über der primären / sekundären Sammlung einzufügen. Weitere Informationen finden Sie unter Arbeiten mit statischem Inhalt.

Wenn Sie Sammlungen in der Galerie bestellen, sollten Sie die Sammlungen wie folgt bestellen:

  • Sammlung 1 - Ausgewählte Sammlung
  • Sammlung 2 - Primärsammlung
  • Sammlung 3 - Sekundäre Sammlung
  • Sammlung 4 - Tracks

Ereignisspuren konfigurieren

Befolgen Sie beim Erstellen der Tracks die folgenden Richtlinien, um sicherzustellen, dass sie in der Erfahrung korrekt angezeigt werden:

  • Die Sammlungsnamen werden in der Vorlage für virtuelle Ereignisse angezeigt. Verwenden Sie daher aussagekräftige Namen
  • Die Titelsammlung ist eine verschachtelte Sammlung. In dieser Sammlung werden keine Videos angezeigt
  • Die Titelsammlung sollte die vierte Sammlung sein
  • Erstellen Sie untergeordnete Sammlungen unter der Titelsammlung für jede Spur
  • Verwenden Sie beim Erstellen der Sammlungen keine Wiedergabelisten
  • Die Position des Tracks-Abschnitts ist in der Konfigurationsdatei und angegeben kann gewechselt werden durch Bearbeiten der breakout_collection Objekt in der Konfigurationsdatei

Konfigurieren der Track-URL

Sobald die Sammlungen erstellt wurden, muss die Konfigurationsdatei aktualisiert werden, damit der Link On-Demand-Sitzungen im Header zur richtigen URL führt. Der für die Titelsammlung verwendete Name (Sammlung 4) wird in der URL verwendet, um auf die Titelseite zuzugreifen. Bestätigen Sie Folgendes, um sicherzustellen, dass die URL korrekt eingestellt ist.

  1. Überprüfen Sie die Konfigurationsdatei und suchen Sie die breakout_collection Objekt.
  2. Das collection Parameter enthält die URL der Titelseite. Beispielsweise:
    
"breakout_collection": {
    "collection": "/category/videos/tracks",
    "insert_target": ".bc-on-demand-section",
    "insert_mode": "after"
},
  3. Der letzte Weg in der collection Die URL sollte den Namen Ihrer Titelsammlung widerspiegeln (Sammlung 4). Wenn Ihre Tracks-Sammlung (Sammlung 4) beispielsweise 'Sessions' heißt, wird der Sammlungsparameter wie folgt aktualisiert:
    
"collection": "/category/videos/sessions",
  4. Klicken Sie zum Überprüfen der Konfiguration in der Kopfzeile auf den Link On-Demand-Sitzungen. Die Track-Sitzung sollte angezeigt werden. Durch Klicken auf einen Titel sollten die Videos in einem Titel geöffnet werden. Wenn ein 404-Fehler angezeigt wird, liegt ein Konfigurationsproblem vor.
