Hinzufügen der Konfigurationsdatei zum Erlebnis

{{ page.description }}

Für die Virtual Event Experience muss eine Konfigurationsdatei in der Site enthalten sein. Die Konfigurationsdatei enthält Site-Einstellungen, Layoutoptionen, Veranstaltungsplan, Lautsprecher und andere Inhalte, die angezeigt werden. Nach dem Erstellen einer neuen virtuellen Ereigniserfahrung sollte zunächst die Konfigurationsdatei hinzugefügt werden

Führen Sie die folgenden Schritte aus, um die Konfigurationsdatei zu einer virtuellen Ereigniserfahrung hinzuzufügen.

  1. Gehen Sie zu folgendem Link:
    https://virtual-events-experience.s3.amazonaws.com/v1/config.txt
  2. Kopieren Sie den Inhalt in die Zwischenablage.
  3. Öffnen Sie das Galerie-Modul.
  4. Bearbeiten Sie die Erfahrung.
  5. Klicken ERSCHEINUNGSBILD UND VERHALTEN> Kopf- und Fußzeile in der linken Navigation.
  6. Fügen Sie die Konfigurationsdatei in das ein Benutzerdefiniertes Header-HTML Feld.
  7. Klicken Sie auf Speichern.

Stellen Sie sicher, dass Sie das Erlebnis erneut veröffentlichen, damit die Änderungen angezeigt werden.

Aktualisieren der policyKey und AssetBaseUrl Werte

Damit der Countdown-Timer in simulativen Videos angezeigt wird, policyKey Wert in der Konfigurationsdatei muss mit einem suchaktivierten Richtlinienschlüssel aktualisiert werden. Der angegebene Wert funktioniert nicht für Ihr Konto. Andernfalls wird kein Countdown für die Videos angezeigt. Schritte zum Abrufen eines suchfähigen Richtlinienschlüssels finden Sie im Entwicklerdokument Codebeispiel: Generieren Sie einen Richtlinienschlüssel. Aktualisieren Sie nach dem Erstellen des neuen Richtlinienschlüssels den policyKey Wert in der Konfigurationsdatei.

Die AssetBaseUrl value path wird zum Laden von Dateien verwendet, die von der Vorlage verwendet werden. Dies kann in den meisten Fällen in Ruhe gelassen werden, es sei denn, es wird ein hohes Maß an Anpassung durchgeführt. Der dort vorhandene Pfadwert sollte entfernt werden.

Nach dem policyKey und AssetBaseUrl Werte in der Konfigurationsdatei aktualisiert wurden, veröffentlichen Sie die Erfahrung erneut.
