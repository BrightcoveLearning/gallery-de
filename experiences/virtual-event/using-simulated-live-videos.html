Simulierte Live-Videos verwenden

{{ page.description }}

Simulativ Kurzform für simuliert live und ist eine Möglichkeit, Endbenutzern VOD-Inhalte wie live zu zeigen. Simulative Inhalte sind für Veranstalter attraktiv, da sie die Voraufzeichnung von Inhalten und umfangreiche Tests vor der Inbetriebnahme ermöglichen. Es reduziert die Risiken, die sich aus potenziellen Ausfalltagen für traditionelle Live-Events ergeben können, die aufgrund von Encoderproblemen oder anderer Logistik auftreten. Simulive Videos zeigen einen Countdown-Timer mit dem Video an, um anzuzeigen, wann es live sein wird. Das folgende Beispiel ist ein simulatives Video in der vorgestellten Sammlung.

Bei Videos in der primären und sekundären Sammlung wird der Countdown-Timer auf der Karte für das Video angezeigt.

Um simulativen Inhalt in einer virtuellen Ereigniserfahrung zu verwenden, muss Folgendes ausgeführt werden:

  • Konfigurieren Sie das Simulive Player Plugin
  • Verbinden Sie den Spieler mit der Erfahrung
  • Erstellen Sie ein simulatives benutzerdefiniertes Feld
  • Konfigurieren Sie simulative Videoeigenschaften
Hinweis: Der Text, der als Teil des Countdown-Timers angezeigt wird, befindet sich im labels Objekt in der Konfigurationsdatei und kann einfach angepasst oder lokalisiert werden.

Konfigurieren des Simulive Player Plugins

Das Simulive Player-Plugin sollte für den Player konfiguriert sein, der in Ihrer Virtual Event Experience verwendet wird. Wenden Sie sich an Ihren Account Manager, um die erforderlichen Informationen zur Konfiguration des Plugins zu erhalten.

  1. Klicken Spieler im Navigationsheader, um das Players-Modul zu öffnen.
  2. Brightcove empfiehlt, einen dedizierten Player für Ihre Virtual Event Experience zu erstellen. Wenn Sie keinen dedizierten Player erstellt haben, klicken Sie auf Neuer Spieler, benennen Sie den Player und klicken Sie dann auf Speichern.
  3. Klicken Sie auf den Link für den Player, den Sie konfigurieren möchten.
  4. Klicken Plugins im linken Navigationsmenü.
  5. Klicken Fügen Sie ein Plugin> Benutzerdefiniertes Plugin hinzu.
  6. Für die Plugin Name eingeben simulivePlugin.
  7. Verwenden Sie für die JavaScript-URL und die CSS-URL die von Ihrem Account Manager bereitgestellten Werte.
  8. Klicken speichern um die Änderungen zu speichern.
Hinweis: Alle an einem Spieler vorgenommenen Änderungen können bis zu 5 Minuten dauern, bis sie bei veröffentlichten Spielern angezeigt werden.

Konfigurieren eines benutzerdefinierten Players

Führen Sie die folgenden Schritte aus, um die Virtual Event Experience für die Verwendung des benutzerdefinierten Players zu konfigurieren.

  1. Öffne das Galerie Modul.
  2. Bearbeiten Sie die Erfahrung.
  3. Klicken VIDEO UND PLAYBACK> Spieler- und Lead-Formular in der linken Navigation.
  4. Wähle aus Verwenden Sie einen ortsspezifischen Player Option und wählen Sie den mit dem Simulive-Plugin konfigurierten Player aus.
  5. Klicken speichern.

Erstellen eines simulativen benutzerdefinierten Felds

In einem benutzerdefinierten Feld werden Datum und Uhrzeit der Verfügbarkeit eines Videos gespeichert. Führen Sie die folgenden Schritte aus, um das simulative benutzerdefinierte Feld zu erstellen.

  1. Klicken Sie in der Navigationskopfzeile auf ADMIN und dann auf Videofelder. Beachten Sie, dass Sie ein Kontoadministrator sein müssen, um auf das ADMIN-Menü zugreifen zu können.
  2. Klicken Benutzerdefiniertes Feld hinzufügen.
  3. Erstellen Sie ein neues benutzerdefiniertes Feld mit den folgenden Werten.
    • Anzeigename - Simuliver Start
    • Interner Name - simulive_start
    • Art - Text
    • Beschreibung - Das erwartete Format ist JJJJ-MM-TTTHH: MM: SS-HH: MM
  4. Klicken Feld speichern.
Informationen zum Erstellen benutzerdefinierter Felder finden Sie unter Benutzerdefinierte Metadatenfelder erstellen.

Simulative Videoeigenschaften konfigurieren

Das benutzerdefinierte Feld Simuliver Start hält das Datum und die Uhrzeit, zu der ein Video verfügbar sein wird. Das Video kann erst angezeigt werden, wenn die Simulive Startzeit erreicht ist. Simulative Videos müssen ebenfalls markiert werden simuliv. Führen Sie die folgenden Schritte aus, um ein Video als simulatives Video zu konfigurieren.

  1. Öffne das Medien Modul.
  2. Klicken Sie auf einen Videotitel, um die Eigenschaften zu öffnen.
  3. Suchen Sie die VIDEO-EIGENSCHAFTEN Abschnitt und klicken Bearbeiten.
  4. Kennzeichnen Sie das Video mit simuliv.
  5. Klicken speichern.
  6. Suchen Sie die BENUTZERDEFINIERTE FELDER Abschnitt und klicken Bearbeiten.
  7. Geben Sie einen Datums- und Zeitwert für die ein Simuliver Start Feld. Das richtige Format des Feldes ist YYYY-MM-DDTHH:MM:SS-HH:MM wo YYYY-MM-DD ist das Jahr, der Monat und das Datum mit den führenden Nullen, HH:MM:SS ist die 24-Stunden-Stunde, Minute und Sekunde mit den entsprechenden Nullen und -HH:MM ist der Zeitzonenversatz (z. B. -04: 00 für UTC-4, +08: 30 für UTC + 8,5 usw.).
    Hinweis: Simulive stützt sich auf die lokale Computerzeit eines bestimmten Benutzers, um zu bestimmen, wann das Ereignis beginnen soll, indem die lokale Computerzeit mit der konfigurierten verglichen wird Simuliver Start Zeit. Aus diesem Grund kann es Fälle geben, in denen ein Ereignis für einige Benutzer früher als erwartet beginnt. Dies liegt normalerweise daran, dass die' Computerzeiten dieser Benutzer ungenau sind.
  8. Klicken speichern.

Aktualisieren des Richtlinienschlüsselwerts

Damit der Countdown-Timer in simulierten Videos angezeigt wird, muss der Richtlinienschlüsselwert in der Konfigurationsdatei mit einem suchfähigen Richtlinienschlüssel aktualisiert werden. Andernfalls wird kein Countdown für die Videos angezeigt. Schritte zum Abrufen eines suchfähigen Richtlinienschlüssels finden Sie unter Codebeispiel: Generieren Sie einen Richtlinienschlüssel. Aktualisieren Sie nach dem Erstellen des neuen Richtlinienschlüssels den policyKey Wert in der Konfigurationsdatei und veröffentlichen Sie die Erfahrung erneut.
