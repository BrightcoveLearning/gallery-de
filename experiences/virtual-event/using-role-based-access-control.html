Verwenden der rollenbasierten Zugriffssteuerung

{{ page.description }}

Wenn Benutzer sich bei Ihrer Virtual Event Experience anmelden müssen, kann die rollenbasierte Zugriffssteuerung verwendet werden, um die Videos zu filtern, die Benutzern basierend auf ihrer Rolle angezeigt werden. Die rollenbasierte Zugriffssteuerung basiert auf einem angegebenen SAML-Attribut für das Benutzerobjekt, das vom Benutzerverwaltungssystem (z. B. Registrierungssystem, SSO) zurückgegeben wird. Regeln werden basierend auf Benutzerrollen und Video Cloud-Tags definiert. Brightcove empfiehlt, dass alle Videos ein Tag haben, um sie für das Ereignis zu identifizieren, sowie zusätzliche Tags für alle rollenbasierten Filter, die durchgeführt werden müssen.

Um die rollenbasierte Zugriffssteuerung zu konfigurieren, legen Sie zunächst fest, wie der Inhalt eingeschränkt wird, und erstellen Sie eine geeignete rollenbasierte Zugriffskonfigurationsdatei. Einige Beispielkonfigurationen finden Sie in den folgenden Abschnitten. Verknüpfen Sie als Nächstes die rollenbasierte Konfigurationsdatei mit der Erfahrung. Zum Schluss kennzeichnen Sie die Videos entsprechend.

Hinweis: Beim Erstellen von Videosammlungen Wiedergabelisten sollte nicht für die Erfassungskriterien verwendet werden, da Wiedergabelisten keine rollenbasierte Filterung der Zugriffssteuerung unterstützen. Verwenden Sie stattdessen Benutzerdefinierte Kriterien.

Erstellen einer rollenbasierten Zugriffskonfigurationsdatei

Die rollenbasierte Zugriffssteuerung wird mithilfe einer separaten rollenbasierten Zugriffskonfigurationsdatei konfiguriert. Das Format der Datei lautet wie folgt:


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {}
}

Die Eigenschaftswerte in der Datei steuern, wie der Zugriff auf die Erfahrung angewendet wird.

  • AdvancedRoleFilter - Bei Einstellung auf True aktiviert die rollenbasierte Zugriffssteuerung
  • defaultIncludeTags - Die Tags, die in allen Videos enthalten sein müssen, damit sie in der Virtual Event Experience angezeigt werden. Auf diese Weise können Sie in der virtuellen Ereigniserfahrung nur eine Teilmenge der Videos im Konto anzeigen.
  • defaultIgnoreTags - Videos mit diesen Tags werden Benutzern, die nicht Teil einer Rolle sind, nicht angezeigt
  • roleAttribute - SAML-Attribut im Benutzerdatensatz, der vom Benutzerverwaltungssystem zurückkommt. Jedes SAML-Attribut kann verwendet werden.
  • valueDelimiter - Trennzeichen zum Trennen mehrerer Rollen (wenn mehrere Rollen zurückgegeben werden können)
  • ignoreByRole - Erweitert die defaultIgnoreTags indem Sie zusätzlich weitere Tags ignorieren, die auf der Rolle des Benutzers basieren. Dies wird verwendet, um noch mehr Videos zu entfernen, die der Benutzer basierend auf seiner Rolle sehen kann.
  • includeByRole - Benutzer mit zugewiesener Rolle sehen nur Videos mit dem angegebenen Tag
  • allowByRole - Erweitert die defaultIncludeTags indem Sie zusätzlich weitere Tags hinzufügen, die auf der Rolle des Benutzers basieren. Dies wird verwendet, um einem Benutzer basierend auf seiner Rolle mehr Videos anzuzeigen. Benutzer mit der angegebenen Rolle sehen Videos, die mit markiert sind defaultIncludeTags sowie die hier aufgeführten Tags.
Einige Beispielkonfigurationsdateien finden Sie in den folgenden Abschnitten.

Einzelne Premium-Rolle

In diesem Szenario können allgemeine Benutzer die meisten Inhalte anzeigen. Eine Teilmenge der mit dem Tag "Premium" gekennzeichneten Inhalte steht jedoch nur Benutzern mit der Rolle "Premium_Benutzer" zur Verfügung. Premium-Benutzer können weiterhin alle Inhalte sehen, die allgemeine Benutzer sehen können. Durch die Verwendung eines "virtual_event" -Tags wird die Site so eingeschränkt, dass nur Inhalte des Kontos angezeigt werden, das speziell für das Ereignis markiert wurde:


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {
          "premium_user": ["premium"]
     }
}

Abgestufter Zugriff auf Inhalte

Ähnlich wie im ersten Szenario, jedoch mit mehreren Ebenen, bei denen jede Ebene Zugriff auf mehr Inhalte erhält und weiterhin alle Inhalte für die unteren Ebenen sehen kann. Benutzerrollen sind "Tier_1", "Tier_2" und "Tier_3", während Benutzer ohne eine dieser Rollen keinen gestuften Inhalt sehen. Tags sind "Tier1", "Tier2" und "Tier3":


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["tier1", "tier2", "tier3"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {},
     "allowByRole": {
          "tier_1": ["tier1"],
          "tier_2": ["tier1", "tier2"],
          "tier_3": ["tier1", "tier2", "tier3"]
     } 
}

Geteilter Inhaltszugriff

In diesem Beispiel sehen die beiden Benutzerrollen nur Videos, die speziell für ihre Rolle markiert wurden. Ein Benutzer ohne Rolle sieht keinen Inhalt für eines der Tags:


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["tag1", "tag2"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {
          "role1": ["tag1"],
          "role2": ["tag2"]
     },
     "allowByRole": {}
}

Segmentierung nach Sprache

Mit dem Split-basierten Ansatz können Inhalte basierend auf einem SAML-Attribut bereitgestellt werden, das die Sprache des Benutzers enthält:


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": [],
     "roleAttribute": "language",
     "valueDelimiter": ",",
     "ignoreByRole": {},
     "includeByRole": {
          "en": ["english"],
          "es": ["spanish"],
          "jp": ["japanese"]
     },
     "allowByRole": {}
}

Beispiel für zusammengeführte Regeln

Dieses erfundene Beispiel zeigt, wie die verschiedenen Optionen kombiniert werden können. Ein allgemeiner Benutzer sieht alle Inhalte mit dem Tag "virtual_event", die nicht mit "premium" gekennzeichnet sind. Ein Benutzer mit der Rolle "Premium_Benutzer" sieht alle allgemeinen Inhalte sowie Premium-Inhalte, jedoch keine Inhalte, die mit "Werbung" gekennzeichnet sind. Ein Benutzer mit der Rolle "trial_user" sieht allgemeine Inhalte und Premium-Inhalte, aber auch Inhalte, die mit "werblich" gekennzeichnet sind. Ein Benutzer mit der Rolle "expired_user" sieht nur Videos, die als Werbe-Tags gekennzeichnet sind:


{
     "advancedRoleFilter": "true",
     "defaultIncludeTags": ["virtual_event"],
     "defaultIgnoreTags": ["premium"],
     "roleAttribute": "role",
     "valueDelimiter": ",",
     "ignoreByRole": {
          "premium_user": ["promotional"]
     },
     "includeByRole": { "expired_user": ["promotional"]
     },
     "allowByRole": {
          "trial_user": ["promotional"],
          "premium_user": ["premium"]
     }
}

Zuweisen eines rollenbasierten Zugriffsprofils

Wenn Sie ein rollenbasiertes Zugriffsprofil erstellt haben, um den Zugriff auf Inhalte zu steuern, verknüpfen Sie es mit der Erfahrung.

  1. Kopieren Sie das Zugriffssteuerungsprofil in die Zwischenablage.
  2. Öffnen Sie das Galerie-Modul.
  3. Bearbeiten Sie die Erfahrung.
  4. Klicken ERSCHEINUNGSBILD UND VERHALTEN> Benutzerdefiniert in der linken Navigation.
  5. Fügen Sie die Zugriffsprofilkonfiguration in das ein Benutzerdefinierte Vorlage JSON Feld.
    Hinweis: Wenn dieses Feld nicht angezeigt wird, wurde das Konto nicht für die rollenbasierte Zugriffssteuerung aktiviert.
  6. Klicken Sie auf Speichern.

Stellen Sie sicher, dass Sie das Erlebnis erneut veröffentlichen, damit die Änderungen angezeigt werden.

Anzeigen von SAML-Attributen

Stellen Sie sicher, dass die SSO-Einstellungen korrekt konfiguriert sind, damit die Benutzer von Ihrem SSO- / Registrierungssystemanbieter auf das Rollenattribut zugreifen können.

  1. Öffnen Sie das Galerie-Modul.
  2. Drücke den Einstellungen Link auf der Galerie-Homepage.
  3. Klicken Sie in der linken Navigation auf Zugriffskontrollprofile. Eine Liste der erstellten Zugriffssteuerungsprofile wird angezeigt.
  4. Klicken Sie auf das Bearbeitungssymbol für das Profil, das von Virtual Event Experience verwendet wird.
  5. Bestätigen Sie, dass die Legen Sie alle Attribute offen Option am unteren Rand der Seite ist aktiviert.
  6. Klicken Sie auf Speichern.
