Vorschau und Veröffentlichung einer Portalerfahrung

{{ page.description }}

Portal-Erlebnisse können während der Entwicklung problemlos in der Vorschau angezeigt werden. Links zur Vorschau der Website können ebenfalls freigegeben werden. Wenn die Site vollständig ist, kann sie veröffentlicht werden, um sie öffentlich zugänglich zu machen.

Vorschau einer Portal-Erfahrung

In der Vorschau einer Erfahrung können Sie sehen, wie die Erfahrung auf Mobil-, Tablet- und Desktopgeräten angezeigt wird. Um eine Vorschau einer Erfahrung anzuzeigen, bearbeiten Sie die Erfahrung und klicken Sie dann auf Vorschau im Gallery Site Editor.

Es gibt zwei Möglichkeiten:

  • Reaktionsschnelle Vorschau - Öffnet die Erfahrung in einem neuen Browserfenster zur Vorschau
  • Get Shareable Link - Bietet eine Vorschau-URL, damit die Vorschau für andere freigegeben werden kann.

Die generierte gemeinsam nutzbare URL ist 72 Stunden gültig.

Hinweis: Wenn Sie ein Sitzungszeitlimit als Teil der Zugriffssteuerungseinstellungen konfiguriert haben, überschreibt das 72-Stunden-Zeitlimit Ihr konfiguriertes Sitzungszeitlimit. Wenn das Erlebnis veröffentlicht wird, wird das Sitzungstimeout (falls vorhanden) durchgesetzt.

Veröffentlichen einer Portal-Erfahrung

Durch das Veröffentlichen einer Portal Experience wird sie den Zuschauern zur Verfügung gestellt und eine statische URL für die Site generiert. Gehen Sie folgendermaßen vor, um eine Erfahrung zu veröffentlichen:

Hinweis: Wenn Sie kürzlich Änderungen an Video-Metadaten oder Wiedergabelisten vorgenommen haben, kann es bis zu 6 Minuten dauern, bis diese Änderungen bei veröffentlichten Erlebnissen angezeigt werden.
  1. Klicken Sie auf den Erfahrungsnamen, um die Site im Site-Editor zu öffnen.
  2. Klicken Veröffentlichen.
  3. Klicken Sie im Dialogfeld Bestätigungsveröffentlichung auf Jetzt veröffentlichen.
  4. Die Erfahrung wird veröffentlicht und die Site-URL wird angezeigt.

Sobald die Erfahrung veröffentlicht wurde, wird die URL als Teil der Site-Details auf der Homepage der Galerie angezeigt.

Veröffentlichen von Änderungen an einer Portal Experience

Wenn Sie Änderungen an einer Erfahrung vornehmen, müssen Sie die Site erneut veröffentlichen, um die Änderungen verfügbar zu machen. Erfahrungen, die erneut veröffentlicht werden müssen, zeigen den Erfahrungsstatus von an Unveröffentlichte Änderungen als Teil der Site-Details.

Befolgen Sie die Schritte unter, um die Änderungen erneut zu veröffentlichen Veröffentlichen einer Site Abschnitt dieses Themas.

Hinweis: Möglicherweise müssen Sie den Browser-Cache und die Cookies löschen, wenn keine Änderungen angezeigt werden.

Indizieren einer Portal-Erfahrung, die von Suchmaschinen indiziert wird

Sobald die Site veröffentlicht ist, können Sie einige Schritte unternehmen, um die Site von Suchmaschinen indizieren zu lassen. Informationen zu diesem Thema finden Sie unter Indizieren einer Galerieportal-Erfahrung.

Portal-Erfahrung offline schalten

Wenn Sie eine Erfahrung offline schalten, ist sie unter ihrer URL nicht mehr verfügbar. Um eine Erfahrung offline zu schalten, klicken Sie auf eine Zeile und dann auf Offline nehmen. Klicken Sie auf, um zu bestätigen, dass die Erfahrung offline geschaltet werden soll Offline nehmen. Um die Erfahrung wieder verfügbar zu machen, muss die Erfahrung erneut veröffentlicht werden. Sobald die Erfahrung offline ist, ist sie nicht mehr über ihre URL erreichbar.

Hinweis: Wenn Sie die Erfahrung offline schalten, wird das Erlebnis nicht gelöscht.
