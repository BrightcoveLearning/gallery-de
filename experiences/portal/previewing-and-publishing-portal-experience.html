    Vorschau und Veröffentlichung eines Portal Experience

    In diesem Thema erfahren Sie, wie Sie die Details für ein In-Page Experience konfigurieren.

    Portal Experiences können während der Entwicklung einfach in der Vorschau angezeigt werden. Links zur Websitevorschau können auch geteilt werden. Wenn die Website fertig ist, kann sie der Öffentlichkeit durch Veröffentlichen zur Verfügung gestellt werden.

    Vorschau auf ein Portalerlebnis

    In der Vorschau eines Erlebnisses können Sie sehen, wie das Erlebnis auf Mobil-, Tablet- und Desktop-Geräten angezeigt wird. Um eine Vorschau eines Erlebnisses anzuzeigen, bearbeiten Sie das Erlebnis und klicken Sie dann im Galerie-Site-Editor auf Vorschau .

    Sie haben zwei Möglichkeiten:

    • Responsive Preview - Öffnet das Erlebnis in einem neuen Browserfenster zur Vorschau
    • Get Shareable Link - Stellt eine Vorschau-URL bereit, damit die Vorschau mit anderen geteilt werden kann.

    Die generierte teilbare URL ist 72 Stunden lang gültig.

    Hinweis: Wenn Sie ein Sitzungstimeout als Teil von Zugriffskontrolleinstellungen konfiguriert haben, überschreibt das Timeout von 72 Stunden das konfigurierte Sitzungstimeout. Wenn das Erlebnis veröffentlicht wird, wird das Sitzungs-Timeout (falls vorhanden) durchgesetzt.

    Veröffentlichen einer Portalerfahrung

    Durch das Veröffentlichen eines Portal Experience wird es für Zuschauer verfügbar und generiert eine statische URL für die Website. Um ein Erlebnis zu veröffentlichen, führen Sie die folgenden Schritte aus:

    Hinweis: Wenn Sie kürzlich Änderungen an Video-Metadaten oder Wiedergabelisten vorgenommen haben, kann es bis zu 6 Minuten dauern, bis diese Änderungen in veröffentlichten Erlebnissen angezeigt werden.
    1. Klicken Sie auf den Namen der Erfahrung, um die Site im Site-Editor zu öffnen.
    2. Klicken Sie auf Publish.
    3. Klicken Sie im Dialogfeld „Publish Confirmation“ auf Publish Now.
    4. Die Erfahrung wird veröffentlicht und die URL der Website wird angezeigt.

    Die Domäne bcvp0rtal unterstützt standardmäßig SSL, weshalb der Zugriff auf Websites auch mit https:// möglich ist. Es sind keine weiteren Einrichtungsschritte oder SSL-Zertifikate erforderlich. Sie haben auch die Möglichkeit, die Website unter Verwendung eigener SSL-Zertifikate zu schützen. Weitere Informationen finden Sie unter Sichern einer Portalerfahrung mit SSL.

    Hinweis: Informationen darüber, wie Sie das Erlebnis auf Ihrer eigenen Domain hosten, finden Sie unter Zuweisen einer benutzerdefinierten Domain zu einer Portal-Erfahrung.

    Sobald das Erlebnis veröffentlicht wurde, wird die URL als Teil der Website-Details auf der Galerie-Homepage angezeigt.

    Veröffentlichen von Änderungen an einem Portal Experience

    Wenn Sie Änderungen an einer Erfahrung vornehmen, müssen Sie die Website erneut veröffentlichen, um die Änderungen verfügbar zu machen. Erfahrungen, die erneut veröffentlicht werden müssen, zeigen einen Erfahrungsstatus von unveröffentlichten Änderungen als Teil der Website-Details an.

    Um die Änderungen erneut zu veröffentlichen, führen Sie die Schritte aus, die im Abschnitt Eine Website veröffentlichen dieses Themas beschrieben sind.

    Hinweis: Möglicherweise müssen Sie den Browser-Cache und die Cookies löschen, wenn keine Änderungen auftreten.

    Ein Portal Experience durch Suchmaschinen indexieren

    Sobald die Website veröffentlicht ist, können Sie einige Schritte unternehmen, um die Website von Suchmaschinen zu indexieren. Informationen zu diesem Thema finden Sie unter Indexierung einer Erfahrung im Gallery Portal.

    Offline ein Portalerlebnis nehmen

    Wenn Sie ein Erlebnis offline nehmen, ist es unter seiner URL nicht mehr verfügbar. Um ein Erlebnis offline zu nehmen, klicken Sie auf eine Zeile und dann auf Offline schalten. Um zu bestätigen, dass die Erfahrung offline geschaltet werden sollte, klicken Sie auf Offline schalten. Um die Erfahrung wieder verfügbar zu machen, muss die Erfahrung erneut veröffentlicht werden. Sobald das Erlebnis offline ist, wird es nicht mehr über seine URL erreichbar sein.

    Hinweis: Wenn Sie das Erlebnis offline nehmen, wird das Erlebnis nicht gelöscht.
