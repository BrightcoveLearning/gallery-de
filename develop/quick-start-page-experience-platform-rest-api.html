Schnellstart: REST-API (In-Page Experience Platform)

Der Schnellstart enthält Beispiele für einige der API-Anforderungen zum Verwalten von In-Page-Erlebnissen sowie eine Beispielschnittstelle für deren Verwendung.

Einführung

Mit der In-Page Experience-API können Sie In-Page-Erlebnisse programmgesteuert verwalten. Dies ist besonders nützlich für CMS-Integrationen, damit Benutzer Verwaltungserfahrungen von ihrem eigenen CMS aus erstellen können, anstatt in Galerie in Studio zu wechseln.

Dieser Schnellstart bietet eine Beispielimplementierung, mit der Benutzer:

  1. Holen Sie sich alle In-Page Experience-Vorlagen für ein Konto
  2. Erstellen Sie eine grundlegende Erfahrung aus der ausgewählten Vorlage
  3. Wählen Sie entweder eine Wiedergabeliste oder eine Liste von Videos aus, um die Erfahrung zu verbessern
  4. Aktualisieren Sie die Erfahrung mit den ausgewählten Videos / Wiedergabelisten
  5. Veröffentlichen Sie die Erfahrung

Proxy-Code

Um Ihre eigene Version der Beispiel-App auf dieser Seite zu erstellen, müssen Sie Ihren eigenen Proxy erstellen und hosten. (Die von Brightcove Learning Services verwendeten Proxys akzeptieren nur Anforderungen von Brightcove-Domänen.) Sie können zwei Versionen unseres Proxy-Codes herunterladen:

  • Dies ist eine allgemeine Version, die erwartet, dass Client-Anmeldeinformationen mit der Anforderung übergeben werden
  • Mit dieser Version können Sie Ihre Client-Anmeldeinformationen im Proxy selbst in den Zeilen 25-26 speichern (empfohlen).
