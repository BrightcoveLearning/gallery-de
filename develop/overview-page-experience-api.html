Einführung

Es gibt zwei APIs, die mit In-Page-Erfahrungen verknüpft sind, die jedoch für sehr unterschiedliche Zwecke verwendet werden:

In-Page Experiences API (REST)

Die In-Page Experiences-API ist eine REST-API, mit der Sie In-Page Experiences programmgesteuert verwalten können. Es ist hauptsächlich für die Integration der Video Cloud-Plattform in Ihre CMS-Anwendung vorgesehen.

Siehe auch die volle API-Referenz für vollständige Details der verfügbaren Anfragen.

In-Page Experiences Client-API

Die In-Page Experiences-Client-API ist eine JavaScript-Bibliothek, mit der Sie das Verhalten der In-Page Experience zur Laufzeit verwalten können, ähnlich wie die Brightcove Player-API zur Steuerung des Verhaltens des Players verwendet wird. Die API ist auf jeder Seite verfügbar, auf der eine oder mehrere eingebettete Brightcove-Erlebnisse ausgeführt werden. Siehe die Überblick: In-Page Experience Client-API für mehr Informationen.

Der Rest dieser Übersicht befasst sich mit der REST-API zum Verwalten Ihrer In-Page-Erfahrungen im Backend.

Eine allgemeine Einführung in In-Page-Erfahrungen finden Sie unter Überblick: Erstellen, Bearbeiten und Veröffentlichen von In-Page-Erlebnissen.

Die folgenden Abschnitte bieten einen Überblick über die 4 Gruppen von REST-API-Anforderungen:

  • Erfahrungen
  • Vorlagen
  • Themen
  • Schriftarten

Erfahrungen

Anfragen nach Erfahrungen fallen in drei Unterkategorien:

Erfahrungen verwalten
Mit diesen Anfragen erhalten Sie eine Liste aller Erfahrungen für das Konto. Sie können auch eine neue Erfahrung erstellen und eine bestimmte Erfahrung abrufen, aktualisieren oder löschen. Schließlich können Sie eine Erfahrung duplizieren, um einen Ausgangspunkt für eine neue zu erstellen.
Erfahrungen veröffentlichen / veröffentlichen
Diese fordern alle auf, eine Liste aller Vorlagen für das Konto oder eine bestimmte Vorlage zu erhalten.
Verwalten Sie den Status für Live-Erlebnisse
Mit diesen beiden Anfragen können Sie entweder den aktuellen Status einer Live-Erfahrung abrufen oder diesen Status ändern (von preEvent zu live , zum Beispiel).

Vorlagen

Vorlagen steuern das allgemeine Erscheinungsbild einer Erfahrung. Benutzerdefinierte Vorlagen können nicht erstellt werden, es gibt jedoch mehrere Standardvorlagen, und im Laufe der Zeit können weitere hinzugefügt werden. Mit der API können Sie eine Liste der Vorlagen abrufen, die Ihrem Konto zur Verfügung stehen, oder eine bestimmte Vorlage abrufen.

Themen

Mithilfe von Designs können Sie eine Vorlage anpassen, um das gewünschte Erscheinungsbild für Ihre Erfahrung zu erzielen. Die API bietet Methoden, um Ihre Themen vollständig zu verwalten, zu erstellen, abzurufen, zu aktualisieren oder zu löschen.

Schriftarten

Sie können Ihre eigenen Schriftarten verwenden. Mit diesen Methoden können Sie Schriftarten hinzufügen, aktualisieren, abrufen oder löschen. Dies sind Web-Schriftarten, die über CSS geladen werden.
