In-Page Experience API-Fehlerreferenz

Dieses Thema enthält eine Fehlerreferenz für die In-Page Experience-API (die Plattform-API).

In-Page Experience API-Fehler

In-Page Experience API-Fehlerreferenz
Error Beispiele Beschreibung
400 BAD_REQUEST 
{
  "status": 400,
  "code": "BAD_REQUEST",
  "message": "No template specified"
}
{
  "status": 400,
  "code": "BAD_REQUEST",
  "message": "Specified template does not exist"
}
{
  "status": 400,
  "code": "BAD_REQUEST",
  "details": [
    "Path `name` is required."
  ]
}
 Normalerweise ein Problem mit dem Anforderungshauptteil, z. B. das Weglassen des template oder ein anderes erforderliches Feld oder Angabe einer Vorlage, die nicht vorhanden ist
401 UNAUTHORIZED 
[
  {
    "error_code": "UNAUTHORIZED",
    "message": "Permission denied."
  }
]
 Entweder wurde mit der Anforderung kein Zugriffstoken gesendet, oder die zum Erstellen des Zugriffstokens verwendeten Clientanmeldeinformationen hatten nicht die richtigen Berechtigungen für die Anforderung
404 NOT_FOUND 
{
  "status": 404,
  "code": "NOT_FOUND",
  "message": "Experience 5bc2624e0f6f340011b not found"
}
 Die Erfahrung, die Sie aktualisieren möchten, ist nicht vorhanden. Es wurde möglicherweise gelöscht oder die Erfahrungs-ID in der Anforderung ist möglicherweise falsch.
