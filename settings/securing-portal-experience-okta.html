Sichern eines Portal-Erlebnisses mit Okta

{{ page.description }}

Wenn eine Erfahrung mit SSO gesichert ist, müssen sich Benutzer authentifizieren, bevor sie auf die Erfahrung zugreifen können.

Hinweis: Die Schritte und Bilder in diesem Thema verwenden die Testversion von Okta SSO.

Bei den Schritten in diesem Thema wurde davon ausgegangen, dass Sie sich für ein Okta-Testkonto angemeldet haben, sich in Ihrem Konto angemeldet haben und auf die Okta-Dashboard-Seite zugreifen können.

Erstellen einer Anwendung

Beginnen Sie mit dem Erstellen einer Anwendung.

  1. Melden Sie sich bei Ihrem Okta-Testkonto an.
  2. Klicken Anwendungen in der oberen Navigation.
  3. Klicken Sie auf der Seite Anwendungen auf Anwendung hinzufügen.
  4. Klicken Neue App erstellen.
  5. Wählen Sie a Plattform von Netz und ein Anmeldemethode von SAML 2.0.
  6. Klicken Erstellen.
  7. Geben Sie ein App Name.
  8. (Optional) Durchsuchen und wählen Sie und App-Logo.
  9. Klicken Nächster.
  10. Stellen Sie die Single Sign-On-URL zu https://auth.bcvp0rtal.com/login/callback.
  11. Für die Zielgruppen-URI (SP-Entitäts-ID) Geben Sie die Okta-URL für Ihre Okta-Startseite ein. Dies wird in der E-Mail des neuen Kontos aufgeführt, die Sie von Okta erhalten haben, als Ihr Testkonto erstellt wurde.
  12. Drücke den <> Vorschau der SAML-Zusicherung Taste. Gültiges XML sollte in einem neuen Browser-Tab angezeigt werden. Bei Fehlern wird eine Meldung angezeigt.
  13. Schließen Sie die Browser-Registerkarte, auf der das XML angezeigt wird, und kehren Sie zur Okta-Seite zurück.
  14. Klicken Sie auf Weiter.
  15. Wählen Sie eine Option für Sind Sie Kunde oder Partner? und dann klicken Fertig.
  16. Bestätigen Sie, dass die Anwendung erstellt wurde und die die Einstellungen Seite erscheint.
  17. Suchen Sie, wo SAML 2.0 erst konfiguriert ist, wenn Sie die Setup-Anweisungen abgeschlossen haben. Klicken Setup-Anweisungen anzeigen.

Die So konfigurieren Sie SAML 2.0 Anweisungen werden in einem neuen Browser-Tab geöffnet. Diese Informationen werden beim Erstellen eines Zugriffssteuerungsprofils in der Brightcove-Galerie benötigt.

Erstellen eines Zugriffssteuerungsprofils

Sie müssen ein Zugriffssteuerungsprofil erstellen, mit dem Sie alle Erfahrungen sichern, die Sie mit SSO sichern möchten.

  1. Melden Sie sich bei Ihrem Brightcove Gallery-Konto an.
  2. Klicken die Einstellungen in der Kopfzeile.
  3. Klicken Sie in der linken Navigation auf Zugriffssteuerungsprofile.
  4. Klicken Profil erstellen.
  5. Geben Sie dem Profil ein Name.
  6. Wähle aus SSO Möglichkeit.
  7. Kehren Sie zu den Anweisungen So konfigurieren Sie SAML 2.0 zurück. Kopiere das Single Sign-On-URL des Identitätsanbieters und füge es in die SAML 2.0-Endpunkt (HTTP) Gebiet.
  8. Kehren Sie zu den Anweisungen So konfigurieren Sie SAML 2.0 zurück. Kopieren Sie den gesamten Wert in die X.509-Zertifikat Feld und fügen Sie es in die X.509-Zertifikat Feld. Stellen Sie sicher, dass Sie die einschließen BEGIN / END CERTIFICATE Linien.
  9. Wenn Sie SSO-Parameter an Gallery zurückgeben, überprüfen Sie die Legen Sie alle Attribute offen Möglichkeit.
  10. Klicken speichern und bestätigen Sie, dass das Profil erstellt wurde.

Gruppenberechtigungen zuweisen

Der letzte Schritt in Okta besteht darin, allen Benutzern in Okta Zugriff auf die Brightcove Engage-Anwendung zu gewähren.

  1. Kehren Sie zum Okta-Dashboard zurück.
  2. Klicken Sie für die von Ihnen erstellte Okta-Anwendung auf Zuordnungen Tab.
  3. Klicken Zuordnen > Gruppen zuordnen.
  4. Klicken Zuordnen neben dem Jeder gruppieren und dann klicken Getan.

Weisen Sie das Zugriffssteuerungsprofil einer Erfahrung zu

Der letzte Schritt besteht darin, das erstellte Zugriffssteuerungsprofil den zu sichernden Erfahrungen zuzuweisen.

  1. Kehren Sie zur Brightcove-Galerie zurück.
  2. Bearbeiten Sie die Erfahrung, die Sie sichern möchten.
  3. Klicken Sie in der linken Navigation auf STANDORTKONFIGURATION> Zugriffskontrolle.
  4. Wähle aus Zugriffssteuerungsprofil Dies ist mit der Okta-SSO-Konfiguration verbunden.
  5. Klicken speichern.
  6. Veröffentlichen die Erfahrung.

Wenn Sie auf die Erfahrung zugreifen, sollten Sie von Okta aufgefordert werden, sich anzumelden.
