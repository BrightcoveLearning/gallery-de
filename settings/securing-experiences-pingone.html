Sichern eines Portalerlebnisses mit PingOne

{{ page.description }}

Wenn eine Erfahrung mit SSO gesichert ist, müssen sich Benutzer authentifizieren, bevor sie auf die Erfahrung zugreifen können.

Hinweis: Die Schritte und Bilder in diesem Thema verwenden die Testversion von PingOne SSO.

Bei den Schritten in diesem Thema wurde davon ausgegangen, dass Sie ein PingOne-Testkonto erstellt und Ihr Konto per E-Mail bestätigt haben.

Erstellen eines Identitätsrepositorys

Beginnen Sie mit der Erstellung eines Identitätsrepositorys.

  1. Melden Sie sich bei Ihrem PingOne-Konto an. Das Instrumententafel Seite wird angezeigt.
  2. Klicken KONFIGURATION in der Kopfzeile. Das die Einstellungen Seite wird geöffnet.
  3. Klicken Stellen Sie eine Verbindung zu einem Identitäts-Repository her.
  4. Klicke auf PINGONVERZEICHNIS und dann klicken Nächster.
  5. Bestätigen Sie, dass das Verzeichnis konfiguriert wurde.
  6. Klicken Nächster.
  7. Für die KARTENATTRIBUTE Behalten Sie die Standardwerte bei und klicken Sie auf Fertig.
Auf der Seite PingOne-Verzeichnis werden die Verzeichnisstatistiken angezeigt.

Erstellen einer Anwendung

Als Nächstes erstellen wir eine Anwendung, die mit SSO gesichert wird.

  1. Klicken ANWENDUNGEN in der Kopfzeile. Das meine Anwendungen Seite wird geöffnet.
  2. Klicken Anwendung hinzufügen.
  3. Wählen Neue SAML-Anwendung.
  4. Geben Sie ein Anwendungsname und Anwendungsbeschreibung.
  5. Wählen Sie a Kategorie (empfehlen Kommunikation).
  6. (Optional) Klicken Veränderung und suchen Sie ein Symbol für die Anwendung und wählen Sie es aus.
  7. Klicken Fahren Sie mit dem nächsten Schritt fort.
  8. Auf der Anwendungskonfiguration Seite, suchen Sie die SAML-Metadaten Feld.
  9. Drücke den Herunterladen Verknüpfung.
Die heruntergeladene Datei wird beim Erstellen eines Zugriffssteuerungsprofils in der Brightcove-Galerie benötigt.

Erstellen eines Zugriffssteuerungsprofils

Sie müssen ein Zugriffssteuerungsprofil erstellen, mit dem Sie alle Erfahrungen sichern, die Sie mit SSO sichern möchten.

  1. Melden Sie sich bei Ihrem Brightcove Gallery-Konto an.
  2. Klicken die Einstellungen in der Kopfzeile.
  3. Klicken Sie in der linken Navigation auf Zugriffssteuerungsprofile.
  4. Klicken Profil erstellen.
  5. Geben Sie dem Profil ein Name.
  6. Überprüf den SSO Möglichkeit.
  7. Suchen Sie die Importieren Sie SSO-Metadaten Abschnitt und dann klicken Datei aussuchen.
  8. Durchsuchen Sie die SAML-Metadatendatei, die Sie von PingOne heruntergeladen haben, und wählen Sie sie aus (im vorherigen Abschnitt).
  9. Vergewissern Sie sich, dass Werte für den SAML 2.0 Endpoint und das X.509-Zertifikat ausgefüllt wurden.
  10. Klicken speichern und bestätigen Sie, dass das Profil erstellt wurde.
  11. Klick auf das Metadaten-URL Feld und speichern Sie die XML-Datei.

Die XML-Datei wird benötigt, um die Anwendungskonfiguration in PingOne abzuschließen.

Abschluss der Anwendungskonfiguration

  1. Kehren Sie zur PingOne-Anwendung zurück.
  2. Auf der Anwendungskonfiguration Abschnitt finden Sie die Metadaten hochladen Feld.
  3. Klicken Sie auf Datei auswählen und wählen Sie die heruntergeladene Datei aus, die Sie nach dem Erstellen des Zugriffssteuerungsprofils in Brightcove Gallery heruntergeladen haben.
  4. Bestätige das Assertion Consumer Service (ACS) und Entitäts-ID Felder wurden nach dem Import mit Werten ausgefüllt.
  5. Klicken Fahren Sie mit dem nächsten Schritt fort.
  6. Auf der SSO-Attributzuordnung Seite, klicken Sie Fahren Sie mit dem nächsten Schritt fort.
  7. Auf der Gruppenzugriff Seite, suchen Sie die Benutzer @ Verzeichnis gruppieren und dann klicken Hinzufügen.
  8. Klicken Fahren Sie mit dem nächsten Schritt fort.
  9. Klicken Sie auf Beenden.
  10. Bestätigen Sie, dass die Anwendung ist Aktiv und aktiviert.

Weisen Sie das Zugriffssteuerungsprofil einer Erfahrung zu

Der letzte Schritt besteht darin, das erstellte Zugriffssteuerungsprofil den zu sichernden Erfahrungen zuzuweisen.

  1. Kehren Sie zur Brightcove-Galerie zurück.
  2. Bearbeiten Sie die Erfahrung, die Sie sichern möchten.
  3. Klicken Sie in der linken Navigation auf STANDORTKONFIGURATION> Zugriffskontrolle.
  4. Wähle aus Zugriffssteuerungsprofil Dies ist mit der PingOne-SSO-Konfiguration verbunden.
  5. Klicken speichern.
  6. Veröffentlichen die Erfahrung.

Wenn Sie auf die Erfahrung zugreifen, sollten Sie von PingOne aufgefordert werden, sich anzumelden.
