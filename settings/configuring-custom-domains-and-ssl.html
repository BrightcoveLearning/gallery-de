Konfigurieren von benutzerdefinierten Domänen und SSL

{{ page.description }}
 

Wenn Sie zum ersten Mal eine Portal-Erfahrung erstellen, wird ihr eine zufällige Domäne mit dem Format zugewiesen <site-slug>.bcvp0rtal.com . Beispielsweise, http://site-148393.bcvp0rtal.com . Wenn die Site veröffentlicht wird, können die Betrachter auf die Site in dieser Domain zugreifen. Galerie bietet die Möglichkeit, Video-Websites in einer von Brightcove bereitgestellten Domain oder in Ihrer eigenen benutzerdefinierten Domain zu veröffentlichen.

Gallery Portal Experiences können auch mehreren URLs zugewiesen werden, sodass Publisher dieselbe Site für verschiedene Funktionen verwenden können, ohne die Site duplizieren zu müssen. Wenn Sie beispielsweise eine Video-Site haben, die Sie in einer Kampagne und als Hauptvideoportal verwenden möchten, können Sie verschiedene URLs mit einem Branding versehen und alle derselben Site zuweisen. Es ist keine Site-Duplizierung erforderlich, und Sie können URLs problemlos löschen, wenn sie nicht mehr benötigt werden.

Hinweis: Wenn Sie mit SSL-Zertifikaten arbeiten, teilen Sie Ihren privaten Schlüssel unter keinen Umständen oder senden Sie ihn an den Brightcove Support.

Verwenden einer von Brightcove bereitgestellten Domain

Mit den von Brightcove bereitgestellten Domains können Sie schnell und einfach eine Domain für Ihre Site erstellen, ohne mit Ihrer IT-Abteilung einen benutzerdefinierten CNAME erstellen oder einen neuen Domainnamen bei einem Registrar registrieren zu müssen. Sie sind "out of the box" verfügbar und können per Knopfdruck implementiert werden. Es gibt drei von Brightcove bereitgestellte Domänen:

  • brightcovegallery.com
  • gallery.video
  • bcvp0rtal.com (das standardmäßig Websites zugewiesen ist)

Alle unterstützen SSL. Um auf die SSL-Site zuzugreifen, fügen Sie einfach https zur URL hinzu. Beispielsweise, https://marquee.brightcovegallery.com

Verwenden einer benutzerdefinierten Domain (Publisher registriert)

Videoseiten können auch auf Ihrer eigenen benutzerdefinierten Domain gehostet werden, die Sie registrieren. Auf diese Weise können Sie die URL mit einem Branding versehen und mit Ihrem aktuellen Website- / URL-Namenskonstrukt konsistent halten. Dazu müssen Sie den Domainnamen registrieren und anschließend einen benutzerdefinierten CNAME-Eintrag bei Ihrem Hosting-Anbieter konfigurieren. Siehe die Abschnitt unten Informationen zum Konfigurieren eines CNAME-Datensatzes.

Erstellen einer benutzerdefinierten Domain

Gehen Sie folgendermaßen vor, um eine benutzerdefinierte Domäne zu erstellen:

  1. Öffnen Sie das Galerie-Modul.
  2. Drücke den die Einstellungen Link auf der Galerie-Homepage.
  3. Klicken Sie in der linken Navigation auf CUSTOM DOMAINS & SSL Verknüpfung.
  4. Klicken Erstellen Sie eine benutzerdefinierte Domain.
  5. Wählen Sie eine zu verwendende Domain-Option aus:
    • brightcovegallery.com - Die Site-Domain hat das Format <your domain>.brightcovegallery.com.
    • gallery.video - Die Site-Domain hat das Format <your domain>.gallery.video .
    • Benutzerdefiniert - Wählen Sie diese Option, um die Site in Ihrer eigenen benutzerdefinierten Domain zu hosten. Wenn Sie diese Option auswählen, müssen Sie auch Konfigurieren Sie einen CNAME-Eintrag mit Ihrem Hosting-Anbieter.
  6. Geben Sie den Domainnamen ein. Eine Meldung zeigt an, ob die Domain gültig oder ungültig ist. Wenn der Domainname ungültig ist, geben Sie einen neuen Domainnamen ein.
  7. Wenn Sie die Site mit einer von Brightcove bereitgestellten Domain veröffentlichen, gibt es eine Option für Erzwingen Sie SSL in dieser Domäne. Prüfen SSL in dieser Domain erzwingen (empfohlen) Aktivieren einer Umleitung von der Nicht-SSL-Version der Site, wenn ein Viewer zu dieser Site wechselt.
  8. Wenn Sie die Site in Ihrer eigenen benutzerdefinierten Domain veröffentlichen, haben Sie die Möglichkeit, die Site mit SSL zu sichern. Überprüf den Verwenden Sie SSL Kontrollkästchen zum Sichern der Site mit SSL. Informationen zur Verwendung von SSL finden Sie unter Sichern einer Portal-Erfahrung mit SSL.
  9. Prüfen Umleitung aktivieren um diese URL zu einer anderen URL umzuleiten. In der Regel wird dies erst überprüft, nachdem diese Domain eine Weile verwendet wurde. Wenn diese Option ausgewählt ist, geben Sie die ein URL umleiten Diese vorhandene URL sollte umgeleitet werden. Prüfen Verwenden Sie eine permanente Umleitung (HTTP-Statuscode 301). um anzuzeigen, dass die Umleitung dauerhaft ist. Wenn eine Suchmaschinenspinne einen 301-Statuscode im Antwortheader einer Webseite findet, versteht sie, dass diese Webseite nicht mehr vorhanden ist, und ersetzt die indizierte URL durch die neue. Diese Option behält Ihren alten Webseitenverkehr und Ihren Seitenrang bei.
  10. Klicken speichern. Eine Liste der konfigurierten Domänen wird angezeigt.
    Hinweis: Bei Verwendung einer von Brightcove bereitgestellten Domäne lautet der SSL-Status immer Unter der Voraussetzung. Dies bedeutet, dass durch das Hinzufügen von https:// am Anfang der Site-URL die Datenübertragung zwischen Ihrem Browser und der Galerie-Site sicher ist.
  11. Erstellen Sie nach Bedarf weitere Domänen.

Um eine Domain zu bearbeiten oder zu löschen, klicken Sie auf das Bearbeiten () oder löschen () Tasten.

Sobald benutzerdefinierte Domänen erstellt wurden, können sie einer Site als Teil der Site-Eigenschaften zugewiesen werden. Weitere Informationen finden Sie unter Zuweisen einer benutzerdefinierten Domäne zu einer Portalerfahrung.

Konfigurieren von CNAME-Einträgen mit Ihrem Hosting-Anbieter

Hinweis: CNAME-Einträge müssen nur konfiguriert werden, wenn einer Site benutzerdefinierte Domänen zugewiesen werden. CNAME-Datensätze müssen NICHT für Websites erstellt werden, die von Brightcove bereitgestellte Domains verwenden.

Die Schritte zum Konfigurieren eines CNAME-Datensatzes (Canonical Name) hängen von Ihrem Hosting-Anbieter ab. Sie müssen für jede benutzerdefinierte Domäne, die Sie verwenden, einen CNAME-Eintrag konfigurieren. Wenn Ihre Site für die Verwendung von SSL konfiguriert ist, sollte der CNAME-Eintrag auf die CloudFront-URL verweisen. Sehen Sichern einer Portal-Erfahrung mit SSL zur Information. Beachten Sie, dass benutzerdefinierte Domänen eine Unterdomäne enthalten müssen, um als CNAME konfiguriert zu werden. Beispiel: video muss vor platziert werden brightcovelearning.com so kann ich meinen CNAME konfigurieren.

Die CNAME-Werte werden als Teil der Einstellungen für benutzerdefinierte Domänen und SSL angezeigt.

In diesem Beispiel würde ich für die benutzerdefinierte Domäne von video.brightcovelearning.com einen CNAME-Datensatz hinzufügen und eine Vielzahl von video und weisen Sie darauf hin d3mkhzp6k6x3hl.cloudfront.net.

Beachten Sie, dass beim direkten Durchsuchen einer Cloudfront-URL der Fehler "Seite nicht gefunden" zurückgegeben wird (404). EIN curl Befehl kann verwendet werden, um die Domänenkonfiguration zu überprüfen. Für dieses Beispiel:

curl -H "Host: video.brightcovelearning.com" https://d3mkhzp6k6x3hl.cloudfront.net

Die Curl-Anweisung sollte den Inhalt der Site-Homepage zurückgeben.

Möglicherweise müssen Sie sich an Ihren Netzwerkadministrator oder Webhosting-Anbieter wenden, um Hilfe bei diesem Schritt zu erhalten. Denken Sie daran, dass es bis zu 48 Stunden dauern kann, bis DNS-Änderungen wirksam werden.

Sichern von Sites mit SSL

Es gibt zwei Möglichkeiten, um eine Portal-Erfahrung zu sichern mit SSL:

  1. Für Websites, die mit dem zugewiesenen Site-Slug (d. h. <site-slug>.bcvp0rtal.com ) veröffentlicht werden oder eine von Brightcove bereitgestellte Domäne verwenden, ist keine Konfiguration zur Unterstützung von https erforderlich. Hinzufügen https: // Durch den Beginn der URL wird die Datenübertragung zwischen Ihrem Browser und der Galerie-Site gesichert. Sie können auch auswählen SSL in dieser Domain erzwingen (empfohlen) Aktivieren einer Umleitung von der Nicht-SSL-Version der Site, wenn ein Viewer zu dieser Site wechselt.
  2. Überprüfen Sie für Websites, die mit Ihrer eigenen benutzerdefinierten Domain gehostet werden, die Verwenden Sie SSL Option bei der Eingabe Ihres Domainnamens. Sie werden dann aufgefordert, Ihre Zertifikatinformationen einzugeben. Weitere Informationen finden Sie unter Sichern einer Portal-Erfahrung mit SSL.
