    Konfigurieren von benutzerdefinierten Domänen und SSL

    In diesem Thema erfahren Sie, wie Sie benutzerdefinierte Domains und SSL für ein Portal Experience konfigurieren.
     

    Wenn Sie zum ersten Mal ein Portal Experience erstellen, wird ihm eine zufällige Domäne mit dem Format von zugewiesen <site-slug>.bcvp0rtal.com . Beispiel: http://site-148393.bcvp0rtal.com . Wenn die Website veröffentlicht wird, können Zuschauer unter dieser Domäne auf die Website zugreifen. Gallery bietet die Möglichkeit, Videoseiten in einer von Brightcove bereitgestellten Domain oder in Ihrer eigenen benutzerdefinierten Domäne zu veröffentlichen.

    Gallery Portal Experiences können auch mehreren URLs zugewiesen werden, sodass Publisher dieselbe Website für verschiedene Funktionen verwenden können, ohne die Website duplizieren zu müssen. Wenn Sie beispielsweise eine Videoseite haben, die Sie in einer Kampagne verwenden möchten, und als Ihr Haupt-Videoportal, können Sie verschiedene URLs markieren und sie alle derselben Website zuweisen. Es ist keine Websiteduplizierung erforderlich, und Sie können URLs leicht löschen, wenn sie nicht mehr benötigt werden.

    Hinweis: Wenn Sie mit SSL-Zertifikaten arbeiten, geben Sie unter keinen Umständen Ihren privaten Schlüssel weiter oder senden Sie ihn an den Brightcove Support.

    Verwenden einer von Brightcove bereitgestellten Domäne

    Mit von Brightcove bereitgestellten Domänen können Sie schnell und einfach eine Domäne für Ihre Website erstellen, ohne mit Ihrer IT-Abteilung einen benutzerdefinierten CNAME erstellen oder einen neuen Domänennamen bei einer Registrierungsstelle registrieren zu müssen. Sie sind „out of the box“ verfügbar und können mit einem Klick auf eine Schaltfläche implementiert werden. Es gibt drei von Brightcove bereitgestellte Domänen:

    • brightcovegallery.com
    • gallery.video
    • bcvp0rtal.com (wird Websites standardmäßig zugewiesen)

    Alle Domänen unterstützen SSL. Fügen Sie der URL einfach „https“ hinzu, um auf die SSL-Website zuzugreifen. Beispiel: https://marquee.brightcovegallery.com

    Verwenden einer benutzerdefinierten Domäne (vom Publisher registriert)

    Videoseiten können auch auf Ihrer eigenen benutzerdefinierten Domain gehostet werden, die Sie registrieren. Sie können für die URL das Branding durchführen und sie an Ihre derzeitigen Benennungen für Websites bzw. URLs anpassen. Hierfür müssen Sie den Domänennamen registrieren und dann einen benutzerdefinierten CNAME-Eintrag mit Ihrem Hosting-Anbieter konfigurieren. Der Abschnitt unten enthält Informationen dazu, wie Sie einen CNAME-Eintrag konfigurieren.

    Verwenden einer benutzerdefinierten Domäne

    Führen Sie die folgenden Schritte aus, um eine benutzerdefinierte Domäne zu erstellen:

    1. Öffnen Sie das Gallery-Modul.
    2. Klicken Sie auf der Galerie-Homepage auf den Link Einstellungen .
    3. Klicken Sie in der linken Navigationsleiste auf den Link CUSTOM DOMAINS & SSL .
    4. Klicken Sie auf Create Custom Domain.
    5. Wählen Sie eine zu verwendende Domainoption aus:
      • brightcovegallery.com - Die Site-Domain hat das Format <your domain>.brightcovegallery.com.
      • gallery.video - Die Site-Domain hat das Format <your domain>.gallery.video .
      • Benutzerdefiniert - Wählen Sie diese Option, um die Website auf Ihrer eigenen benutzerdefinierten Domain zu hosten. Beim Auswählen dieser Option müssen Sie auch einen CNAME-Eintrag mit Ihrem Hosting-Anbieter konfigurieren.
    6. Geben Sie den Domänennamen ein. Eine Meldung mit dem Hinweis, ob die Domäne gültig oder ungültig ist, wird angezeigt. Wenn der Domainname ungültig ist, geben Sie einen neuen Domainnamen ein.
    7. Wenn Sie die Website mit einer von Brightcove bereitgestellten Domäne veröffentlichen, besteht die Möglichkeit, SSL für diese Domain durchzusetzen. Aktivieren Sie SSL auf dieser Domain durchsetzen (empfohlen) , um eine Weiterleitung von der Nicht-SSL-Version der Site zu aktivieren, wenn ein Betrachter darauf eingeht.
    8. Wenn Sie die Website in Ihrer eigenen benutzerdefinierten Domäne veröffentlichen, können Sie die Option zum Schützen der Website per SSL nutzen. Aktivieren Sie das Kontrollkästchen Use SSL, um die Website per SSL zu schützen. Informationen zur Verwendung von SSL finden Sie unter Sichern einer Portalerfahrung mit SSL.
    9. Aktivieren Sie Umleitung aktivieren, um diese URL an eine andere URL umzuleiten. In der Regel wird dies erst überprüft, nachdem diese Domain eine Weile benutzt wurde. Wenn diese Option ausgewählt ist, geben Sie die Umleitungs-URL ein, zu der diese vorhandene URL umgeleitet werden soll. Aktivieren Sie Dauerhafte Weiterleitung verwenden (HTTP-Statuscode 301) , um anzugeben, dass die Umleitung dauerhaft ist. Wenn eine Suchmaschinen-Spinne im Antwortheader einer Webseite einen 301-Statuscode findet, versteht sie, dass diese Webseite nicht mehr existiert, und ersetzt die indexierte URL durch die neue. Diese Option behält Ihren alten Webseitenverkehr und Ihren Seitenrang bei.
    10. Klicken Sie auf Save. Eine Liste mit konfigurierten Domänen wird angezeigt.
      Hinweis: Bei Verwendung einer von Brightcove bereitgestellten Domäne lautet der SSL-Status immer Provided. Hiermit wird angegeben, dass die Datenübertragung zwischen Ihrem Browser und der Gallery-Website geschützt wird, indem am Anfang der Website-URL https:// hinzugefügt wird.
    11. Erstellen Sie je nach Bedarf weitere Domänen.

    Klicken Sie auf die Schaltfläche „Edit“ () oder „Delete“ (), um eine Domäne zu bearbeiten bzw. zu löschen.

    Nachdem benutzerdefinierte Domänen erstellt wurden, können sie einer Website im Rahmen der Websiteeigenschaften zugewiesen werden. Weitere Informationen finden Sie unter Zuweisen einer benutzerdefinierten Domäne zu einem Portal Experience.

    Konfigurieren eines CNAME-Eintrags mit Ihrem Hosting-Anbieter

    Hinweis: CNAME-Einträge müssen nur konfiguriert werden, wenn einer Website benutzerdefinierte Domänen zugewiesen werden. CNAME-Einträge müssen NICHT für Websites erstellt werden, für die von Brightcove bereitgestellte Domänen verwendet werden.

    Die Schritte zum Konfigurieren eines CNAME-Eintrags („Canonical Name“) hängen von Ihrem Hosting-Anbieter ab. Sie müssen für jede benutzerdefinierte Domäne, die Sie verwenden möchten, einen CNAME-Eintrag konfigurieren. Wenn Ihre Website für die Verwendung von SSL konfiguriert ist, sollte der CNAME-Eintrag auf die CloudFront-URL verweisen. Weitere Informationen finden Sie unter Sichern einer Portalerfahrung mit SSL . Beachten Sie, dass benutzerdefinierte Domains eine Subdomain enthalten müssen, damit sie als CNAME ex: konfiguriert werden video müssen, brightcovelearning.com damit ich meinen CNAME konfigurieren kann.

    Die CNAME-Werte werden als Teil der Einstellungen vom Typ „Custom Domains & SSL“ angezeigt.

    In diesem Beispiel für die benutzerdefinierte Domäne von video.brightcovelearning.com ist es ratsam, einen CNAME-Eintrag und einen Host von video hinzuzufügen und dafür auf d3mkhzp6k6x3hl.cloudfront.net zu verweisen.

    Beachten Sie, dass das direkte Browsen zu einer Cloudfront-URL einen Fehler „Seite nicht gefunden“ (404) zurückgibt. Ein curl Befehl kann verwendet werden, um die Domänenkonfiguration zu überprüfen. Für dieses Beispiel:

    curl -H "Host: video.brightcovelearning.com" https://d3mkhzp6k6x3hl.cloudfront.net

    Die curl-Anweisung sollte den Inhalt der Homepage der Website zurückgeben.

    Möglicherweise müssen Sie sich an Ihren Netzwerkadministrator oder Webhosting-Anbieter wenden, um Hilfe bei diesem Schritt zu erhalten. Beachten Sie, dass es bis zu 48 Stunden dauern kann, bis DNS-Änderungen wirksam werden.

    Schützen von Websites per SSL

    Es gibt zwei Möglichkeiten, ein Portal Experience mit SSL zu sichern:

    1. Für Websites, die mit der zugewiesenen Kurzbeschreibung (d. h. <site-slug>.bcvp0rtal.com ) veröffentlicht werden oder eine von Brightcove bereitgestellte Domäne nutzen, ist zur Unterstützung von „https“ keine Konfiguration erforderlich. Indem am Anfang der URL https:// hinzugefügt wird, wird die Datenübertragung zwischen Ihrem Browser und der Gallery-Website geschützt. Sie können auch SSL erzwingen für diese Domain auswählen (empfohlen) , um eine Weiterleitung von der Nicht-SSL-Version der Site zu aktivieren, wenn ein Betrachter darauf eingeht.
    2. Für Websites, die mit Ihrer eigenen benutzerdefinierten Domäne gehostet werden, aktivieren Sie beim Eingeben Ihres Domänennamens die Option Use SSL. Sie werden dann aufgefordert, Ihre Zertifikatinformationen einzugeben. Weitere Informationen finden Sie unter Sichern einer Portalerfahrung mit SSL.
