    Sichern eines Portalerlebnisses mit Auth0

    In diesem Thema erfahren Sie, wie Sie Portal Experiences mit Auth0 SSO sichern können.

    Wenn ein Erlebnis mit SSO gesichert wird, müssen sich Benutzer authentifizieren, bevor sie auf das Erlebnis zugreifen können.

    Hinweis: Die Schritte und Bilder in diesem Thema verwenden die Testversion von Auth0.

    Die Schritte in diesem Thema gehen davon aus, dass Sie ein Auth0-Testkonto erstellt und Ihr Konto per E-Mail bestätigt haben.

    Erstellen eines Zugriffssteuerungsprofils

    Sie müssen ein Zugriffssteuerungsprofil erstellen, das verwendet wird, um alle Erlebnisse zu sichern, die Sie mit SSO sichern möchten.

    1. Melden Sie sich bei Video Cloud Studioan.
    2. Öffnen Sie das Galerie-Modul .
    3. Klicken Sie in der Kopfzeile auf Einstellungen .
    4. Klicken Sie in der linken Navigation auf Access Control Profiles.
    5. Klicken Sie auf Create Profile.
    6. Geben Sie dem Profil unter Name einen Namen.
    7. Prüfen Sie die SSO-Option .

    Um das Access Control Profile zu vervollständigen, benötigen Sie einige Werte, die im Rahmen des Auth0-Anwendungserstellungsprozesses bereitgestellt werden. Nachdem wir eine neue Auth0-Anwendung erstellt haben, können wir das Erstellen dieses Profils abschließen.

    Erstellen einer Anwendung

    Als Nächstes müssen Sie eine Anwendung erstellen, die mit Auth0 gesichert wird. Im Rahmen des Anwendungserstellungsprozesses werden das Signaturzertifikat und die SAML-Protokoll-URL vom im vorherigen Abschnitt erstellten Access Control-Profil verwendet.

    1. Melden Sie sich bei Ihrem Auth0-Konto an.
    2. Klicken Sie auf der Seite „ Erste Schritte “ in der linken Navigationsleiste auf Anwendungen.
    3. Klicken Sie auf der Seite „ Anwendungen “ auf + ANWENDUNG ERSTELLEN.
    4. Geben Sie einen Namen für die Anwendung ein.
    5. Wählen Sie einen Anwendungstyp für reguläre Webanwendungenaus.
    6. Klicken Sie auf ERSTELLEN.
    7. Klicken Sie unter dem Anwendungsnamen auf den Link Einstellungen .
    8. (Optional) Suchen Sie das Feld Anwendungslogo und geben Sie eine URL zu Ihrem Anwendungslogo ein. Dies führt dazu, dass Ihr Logo im Vergleich zum Auth0-Logo auf der Anmeldeseite angezeigt wird.
    9. Bestätigen Sie, dass die Token Endpoint Authentication Method auf POSTfestgelegt ist.
    10. Fügen Sie im Feld Rückruf-URLs zulassen die folgende URL hinzu: https://auth.bcvp0rtal.com/login/callback
    11. Klicken Sie unten auf der Seite auf den Link Erweiterte Einstellungen anzeigen .
    12. Klicken Sie auf den Link Zertifikate .
    13. Kopieren Sie das Signaturzertifikat in die Zwischenablage.
    14. Kehren Sie zu Gallery und dem Access Control-Profil zurück, das Sie im vorherigen Abschnitt erstellt haben.
    15. Fügen Sie das Signaturzertifikat in das Feld X.509-Zertifikat ein.
    16. Entfernen Sie die Zeilen —BEGIN CERTIFICATE— und —END CERTIFICATE— aus dem X.509-Zertifikat .
    17. Kehren Sie zu Auth0 zurück.
    18. Klicken Sie auf den Link Endpoints .
    19. Suchen Sie die SAML-Protokoll-URL und kopieren Sie sie in die Zwischenablage.
    20. Kehren Sie zu Gallery und dem Access Control-Profil zurück, das Sie im vorherigen Abschnitt erstellt haben.
    21. Fügen Sie die SAML-Protokoll-URL in das Feld SAML 2.0 Endpoint (HTTP) ein.
    22. Aktivieren Sie die Option Alle Attribute offenlegen am Ende der Seite.
    23. Klicken Sie auf Speichern , um das Access Control Profile zu speichern.
    24. Kehren Sie zu Auth0 zurück.
    25. Klicken Sie oben auf der Seite unter dem Namen der Anwendung auf den Link Addons .
    26. Aktivieren Sie die Option SAML2 WEB APP .
    27. Fügen Sie im Feld Anwendungsrückruf-URL die folgende URL hinzu: https://auth.bcvp0rtal.com/login/callback
      Hinweis: Dieser Wert kann bereits ausgefüllt sein.
    28. Scrollen Sie zum Ende des Dialogs und klicken Sie auf AKTIVIEREN.
    29. Schließen Sie den Dialog.

    Stellen Sie sicher, dass Sie die Änderungen an den Anwendungseinstellungen speichern.

    Erstellen eines Benutzers in Auth0

    In diesem Thema wird ein Testkonto von Auth0 verwendet. Standardmäßig werden keine Benutzer erstellt. Möglicherweise müssen Sie einen Benutzer hinzufügen, um den Testprozess abzuschließen.

    1. Kehren Sie zu Auth0 zurück.
    2. Klicken Sie im linken Navigationsbereich auf Benutzer & Rollen > Benutzer.
    3. Klicken Sie auf + BENUTZER ERSTELLEN.
    4. Geben Sie die Benutzerinformationen ein und klicken Sie auf ERSTELLEN.

     

    Weisen Sie das Access Control-Profil einem Erlebnis zu

    Der letzte Schritt besteht darin, das erstellte Zugriffssteuerungsprofil den Erlebnissen zuzuweisen, die Sie sichern möchten.

    1. Kehren Sie zu Gallery zurück
    2. Bearbeiten Sie das Erlebnis, das Sie sichern möchten.
    3. Klicken Sie im linken Navigationsbereich auf SITE CONFIGURATION > Access Control.
    4. Wählen Sie das Zugriffssteuerungsprofil aus, das mit der Auth0-SSO-Konfiguration verknüpft ist.
    5. Klicken Sie auf Save.
    6. Veröffentlichen Sie die Erfahrung.

    Wenn Sie auf das Erlebnis zugreifen, sollten Sie von Auth0 aufgefordert werden, sich anzumelden.
