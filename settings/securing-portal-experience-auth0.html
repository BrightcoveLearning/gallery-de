Sichern einer Portal-Erfahrung mit Auth0

{{ page.description }}

Wenn eine Erfahrung mit SSO gesichert ist, müssen sich Benutzer authentifizieren, bevor sie auf die Erfahrung zugreifen können.

Hinweis: Die Schritte und Bilder in diesem Thema verwenden die Testversion von Auth0.

Bei den Schritten in diesem Thema wurde davon ausgegangen, dass Sie ein Auth0-Testkonto erstellt und Ihr Konto per E-Mail bestätigt haben.

Erstellen eines Zugriffssteuerungsprofils

Sie müssen ein Zugriffssteuerungsprofil erstellen, mit dem Sie alle Erfahrungen sichern, die Sie mit SSO sichern möchten.

  1. Melden Sie sich bei Video Cloud Studio an.
  2. Öffne das Galerie Modul.
  3. Klicken die Einstellungen in der Kopfzeile.
  4. Klicken Sie in der linken Navigation auf Zugriffssteuerungsprofile.
  5. Klicken Profil erstellen.
  6. Geben Sie dem Profil ein Name.
  7. Überprüf den SSO Möglichkeit.

Um das Zugriffssteuerungsprofil zu vervollständigen, benötigen Sie einige Werte, die im Rahmen des Erstellungsprozesses der Auth0-Anwendung bereitgestellt werden. Nachdem Sie eine neue Auth0-Anwendung erstellt haben, können Sie die Erstellung dieses Profils abschließen.

Erstellen einer Anwendung

Als Nächstes müssen Sie eine Anwendung erstellen, die mit Auth0 gesichert wird. Im Rahmen der Anwendungserstellung werden das Signaturzertifikat und die SAML-Protokoll-URL vom im vorherigen Abschnitt erstellten Zugriffssteuerungsprofil verwendet.

  1. Melden Sie sich bei Ihrem Auth0-Konto an.
  2. Auf der Einstieg Seite, klicken Sie in der linken Navigation auf Anwendungen.
  3. Auf der Anwendungen Seite, klicken Sie + ANWENDUNG ERSTELLEN.
  4. Geben Sie ein Name für die Anwendung.
  5. Wählen Sie einen Anwendungstyp von Regelmäßige Webanwendungen.
  6. Klicken ERSTELLEN.
  7. Drücke den die Einstellungen Link unter dem Anwendungsnamen.
  8. (Optional) Suchen Sie die Anwendungslogo Feld und geben Sie eine URL zu Ihrem Anwendungslogo ein. Dadurch wird Ihr Logo auf der Anmeldeseite im Vergleich zum Auth0-Logo angezeigt.
  9. Bestätigen Sie, dass die Token-Endpunkt-Authentifizierungsmethode ist eingestellt auf POST.
  10. In dem Rückruf-URLs zulassen Fügen Sie im Feld die folgende URL hinzu: https://auth.bcvp0rtal.com/login/callback
  11. Klicken Sie unten auf der Seite auf Erweiterte Einstellungen anzeigen Verknüpfung.
  12. Drücke den Zertifikate Verknüpfung.
  13. Kopiere das Unterzeichnungszertifikat in die Zwischenablage.
  14. Kehren Sie zur Galerie und zum Zugriffssteuerungsprofil zurück, die Sie im vorherigen Abschnitt erstellt haben.
  15. Fügen Sie das Signaturzertifikat in das Feld ein X.509-Zertifikat Feld.
  16. Entferne das --- BEGIN CERTIFICATE --- und --- ENDZERTIFIKAT --- Zeilen aus dem X.509-Zertifikat Feld.
  17. Zurück zur Seite Auth0.
  18. Drücke den Endpunkte Verknüpfung.
  19. Suchen Sie die SAML-Protokoll-URL und kopieren Sie es in die Zwischenablage.
  20. Kehren Sie zur Galerie und zum Zugriffssteuerungsprofil zurück, die Sie im vorherigen Abschnitt erstellt haben.
  21. Fügen Sie die SAML-Protokoll-URL in das SAML 2.0-Endpunkt (HTTP) Gebiet.
  22. Überprüf den Legen Sie alle Attribute offen Option am Ende der Seite.
  23. Klicken speichern um das Zugriffssteuerungsprofil zu speichern.
  24. Zurück zur Seite Auth0.
  25. Klicken Sie oben auf der Seite unter dem Anwendungsnamen auf Addons Verknüpfung.
  26. Aktivieren Sie die SAML2 WEB APP Möglichkeit.
  27. In dem Rückruf-URL der Anwendung fügen Sie die folgende URL hinzu: https://auth.bcvp0rtal.com/login/callback
    Notiz: Dieser Wert ist möglicherweise bereits ausgefüllt.
  28. Scrollen Sie zum Ende des Dialogfelds und klicken Sie auf AKTIVIEREN.
  29. Schließen Sie den Dialog.

Stellen Sie sicher, dass Sie die Änderungen an den Anwendungseinstellungen speichern.

Erstellen eines Benutzers in Auth0

In diesem Thema wird ein Testkonto von Auth0 verwendet. Standardmäßig werden keine Benutzer erstellt. Möglicherweise müssen Sie einen Benutzer hinzufügen, um den Testvorgang abzuschließen.

  1. Zurück zur Seite Auth0.
  2. Klicken Sie in der linken Navigation auf Benutzer & Rollen> Benutzer.
  3. Klicken + BENUTZER ERSTELLEN.
  4. Geben Sie die Benutzerinformationen ein und klicken Sie auf ERSTELLEN.

 

Weisen Sie das Zugriffssteuerungsprofil einer Erfahrung zu

Der letzte Schritt besteht darin, das erstellte Zugriffssteuerungsprofil den zu sichernden Erfahrungen zuzuweisen.

  1. Zurück zur Galerie.
  2. Bearbeiten Sie die Erfahrung, die Sie sichern möchten.
  3. Klicken Sie in der linken Navigation auf STANDORTKONFIGURATION> Zugriffskontrolle.
  4. Wähle aus Zugriffssteuerungsprofil Dies ist mit der Auth0-SSO-Konfiguration verbunden.
  5. Klicken speichern.
  6. Veröffentlichen die Erfahrung.

Wenn Sie auf die Erfahrung zugreifen, sollten Sie von Auth0 aufgefordert werden, sich anzumelden.
