Konfigurieren der einmaligen Anmeldung mit Active Directory-Verbunddiensten

{{ page.description }}

Bevor Sie anfangen

Führen Sie die folgenden Schritte zur Einrichtung aus:

  1. Aktivieren Sie RelayState für die iDP-initiierte Anmeldung, indem Sie den Anweisungen folgen Hier. Der Speicherort hängt von der verwendeten ADFS-Version ab.
  2. Starten Sie den ADFS-Dienst neu, damit die Änderungen übernommen werden.

Hinzufügen eines Galeriezugriffssteuerungsprofils

  1. Öffnen Verwaltungswerkzeuge Klicken Sie im Windows-Startmenü oder in der Systemsteuerung auf AD FS-Verwaltung Anwendung.
  2. Öffnen Dienstleistungen> Zertifikate im linken Explorer-Bereich.
  3. Doppelklicken Sie auf Token-Signierung Zertifikat, das Sie verwenden möchten.
  4. Klick auf das Einzelheiten Tab und klicken In Datei kopieren…
  5. Klicken Nächster Wählen Sie dann im Exportassistenten die Option aus Base-64-codiertes X.509 (.CER) Möglichkeit.
  6. Speichern Sie die Zertifikatdatei in Ihrem lokalen Dateisystem und öffnen Sie sie dann in einem Texteditor. Dies ist das Zertifikat, das zum Einrichten des Zugriffssteuerungsprofils für die Galerie verwendet wird.
  7. Öffnen Sie das Galerie-Modul.
  8. Klicken die Einstellungen.
  9. Klicken Zugriffssteuerungsprofile in der linken Navigation.
  10. Klicken Profil erstellen.
  11. Gib es ein Name und dann auswählen SSO - Erfordert einen Benutzernamen und ein Passwort für den Zugriff.
  12. In dem SAML 2.0-Endpunkt (HTTP) Geben Sie im Feld die URL zur iDP-initiierten Anmeldeseite für Ihren ADFS-Server ein. Dies endet normalerweise in IdpInitiatedSignOn.aspx. Zum Beispiel, wenn der SP-initiierte Anmeldelink ist https://sso-test.brightcove.com/adfs/ls ist die von iDP initiierte Anmeldeseite https://sso-test.brightcove.com/adfs/ls/IdpInitiatedSignOn.aspx.
  13. Überprüf den Mein SSO-System ist ADFS (Active Directory Federation Services). Möglichkeit.
  14. Kopieren Sie das in Schritt 6 gespeicherte Zertifikat und fügen Sie es in das ein X.509-Zertifikat Feld.
  15. Klicken speichern um das Zugriffssteuerungsprofil zu speichern. Das neue Zugriffssteuerungsprofil sollte in der Liste der Profile angezeigt werden.
  16. Öffnen Sie wieder in der AD FS-Verwaltungsanwendung Vertrauen auf Parteivertrauensstellungen.
  17. Klicken Vertrauen der vertrauenden Partei hinzufügen… in der rechten Tafel.
  18. Klicken Start im Assistenten und fügen Sie dann die Metadaten-URL das wurde in Schritt 15 in der angezeigt Föderations-Metadatenadresse Feld.
  19. Klicken Nächster und geben Sie dann Ihrem Vertrauen der vertrauenden Partei einen Namen.
  20. Klicken Sie weiter Nächster bis zum Fertig Schritt. Deaktivieren Sie das Kontrollkästchen Konfigurieren Sie die Richtlinien zur Ausstellung von Ansprüchen für diese Anwendung Kontrollkästchen. Klicken Schließen.

Sie sollten nun in der Lage sein, dem erstellten Zugriffssteuerungsprofil eine Portalerfahrung zuzuordnen. Nachdem die Erfahrung veröffentlicht wurde, sollte die Navigation dazu den ADFS-Anmeldeworkflow durchlaufen.
